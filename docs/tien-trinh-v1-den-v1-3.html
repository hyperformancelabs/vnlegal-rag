VNLegal RAG — Bạn đã làm gì từ v1 đến v1.3?

Tài liệu này tổng hợp toàn bộ mã nguồn trong repo và giải thích bằng tiếng Việt cách dự án phát triển qua từng phiên bản: từ nền tảng dữ liệu + huấn luyện mô hình (v1), tinh chỉnh pipeline truy xuất (v2), đến phiên bản chuẩn hoá cho demo (v1.3).

Đồ án: Hệ thống hỏi đáp pháp luật tiếng Việt theo hướng RAG — định tuyến chủ đề (TextCNN) + truy xuất ngữ nghĩa (Siamese) + sinh câu trả lời (Groq, tuỳ chọn).
Xem thêm: giai-thich-du-an.html (sơ đồ, biểu đồ, API demo chi tiết).

Tóm tắt một dòng

v1 — Nền tảng

Chuẩn hoá dữ liệu 8 lớp, train TextCNN + 4 biến thể Siamese, chọn LSTM + Cosine (MRR 0.702).

v2 — Tuning retrieval

Cùng model v1, thêm Hybrid TF-IDF + Siamese và quét topic_topk / alpha.

v1.3 — Demo & production

Gộp 3 lớp hiếm → other, train lại trên data_ready_v1_3, demo BiLSTM 1L + UI web.

Dòng thời gian phát triển

v1 — Giai đoạn khởi tạo

Dữ liệu + mô hình trong thư mục data/ và model/

  • load.ipynb → eda.ipynb → prepare-data.py xuất data/data_ready/ (8 macro_domain).
  • Tokenizer chuẩn model/tokenizer.py + embedding dùng chung build_shared_embedding.py.
  • Huấn luyện TextCNN và 4 notebook Siamese (LSTM/BiLSTM × Cosine/Euclidean).
  • Benchmark: Siamese LSTM truyền thống + Triplet Cosine thắng (Recall@1 = 0.6088, MRR = 0.7020).
  • Pipeline suy luận: model/test-pipeline.ipynb — TextCNN top-2 topic → lọc corpus → Siamese xếp hạng.
v2 — Tinh chỉnh pipeline (không đổi tập train chính)

model/test-pipeline-v2.ipynb

  • Giữ artifact model v1, mở rộng không gian cấu hình đánh giá retrieval.
  • Quét topic_topk ∈ {3, 4} (v1 cố định top-2).
  • Thêm pipeline Hybrid: score = α·TF-IDF_norm + (1-α)·Siamese_norm với α ∈ {0, 0.25, 0.5, 0.75, 1}.
  • Kết quả ghi trong docs/so-sanh-pipeline.md: Hybrid (top-4, α=0.5) gần bằng TF-IDF full corpus (MRR 0.6766 vs 0.6768).
v1.1 / v1.2 — Trung gian (tham chiếu trong code)

Chuẩn bị cho v1.3

  • Thư mục pipeline_v1.1/ không còn trong repo; notebook v1.3 ghi chú kế thừa train-textcnn_v2.ipynb từ v1.1.
  • data/data_ready_v1_2/ được nhắc trong cây thư mục — phiên bản dữ liệu trung gian trước khi gộp lớp hiếm.
  • pipeline_v1.3/prepare_data.py dùng cùng StratifiedGroupKFold như v1.1, chỉ khác bước gộp nhãn.
v1.3 — Phiên bản hiện hành

Thư mục pipeline_v1.3/ + demo web

  • prepare_data.py: gộp 3 lớp macro_domain hiếm nhất trên train → other (còn 6 lớp).
  • Train lại TextCNN + 3 biến thể Siamese (LSTM uni 1L, BiLSTM 1L, BiLSTM 2L) trên data/data_ready_v1_3/.
  • test-pipeline-v1_3.ipynb: benchmark end-to-end — BiLSTM 1L + TextCNN top-3 dẫn đầu (MRR 0.6891 trên 200 query test).
  • demo_siamese_bilstm1l_ui.py: server HTTP + giao diện hỏi đáp + Groq RAG (tuỳ chọn).

Giai đoạn v1 — Bạn đã xây nền tảng gì?

Đây là giai đoạn nghiên cứu và chọn mô hình. Mục tiêu: có pipeline dữ liệu tái lập được, huấn luyện bộ phân loại miền pháp lý và mô hình truy xuất, rồi chọn một cấu hình Siamese tốt nhất.

1. Pipeline dữ liệu (thư mục data/)

data/raw (vnlegal_corpus, vnlegal_qa) ↓ load.ipynb — tạo passage_id, qa_id, gán macro_domain heuristic từ doc_name ↓ eda.ipynb — làm sạch, phân tích null/trùng/phân phối → data/processed/ ↓ prepare-data.py — dedup, join doc_name, StratifiedGroupKFold theo doc_name → data/data_ready/ (qa_train|val|test, corpus_*, label_maps.json — 8 lớp)

Điểm quan trọng: chia tập theo doc_name (group) để cùng một văn bản pháp luật không xuất hiện ở cả train và test — tránh “học thuộc” văn bản thay vì học ngữ nghĩa.

2. Tokenizer & embedding dùng chung

File model/tokenizer.py — mọi mô hình dùng chung:

  • simple_tokenize: lowercase + regex \w+ (Unicode, giữ tiếng Việt có dấu).
  • encode_text / encode_with_mask cho TextCNN và Siamese.
  • build_vocab → tokenizer_vocab.json + embedding.pt (khởi tạo N(0, 0.02²), PAD = 0).

3. TextCNN — phân loại macro_domain

Notebook model/train-textcnn.ipynb. Kiến trúc Kim (2014): Conv1D kernel {3,4,5}, 100 filters, max-over-time pool, dropout, softmax 8 lớp.

Vai trò trong RAG: từ câu hỏi người dùng → dự đoán 1–2 chủ đề pháp lý → thu hẹp corpus trước khi Siamese xếp hạng (giảm chi phí tính toán).

4. Siamese — bốn thí nghiệm, một người thắng

5. Pipeline truy xuất v1 (test-pipeline.ipynb)

Câu hỏi người dùng → TextCNN: lấy top-2 macro_domain → Lọc corpus chỉ giữ passage thuộc 2 miền đó → SiameseLSTM: encode query + từng passage → cosine similarity → Trả top-K passage (Recall@K, MRR, nDCG trên qa_test) (+ baseline TF-IDF full corpus và TF-IDF + lọc topic)

Pipeline v2 — Tinh chỉnh retrieval (không train lại từ đầu)

Notebook model/test-pipeline-v2.ipynb dùng cùng checkpoint đã train ở v1, nhưng mở rộng cách kết hợp TF-IDF và Siamese khi xếp hạng.

Cách chạy theo phiên bản

Chạy pipeline v1.3 (khuyến nghị — demo)

# Từ thư mục gốc repo
python -m pip install -r requirements.txt

python pipeline_v1.3/prepare_data.py --merge-bottom-k 3
python pipeline_v1.3/build_shared_embedding.py --max-vocab 8000 --embed-dim 200

# Train (notebook hoặc Kaggle) — cần GPU
# pipeline_v1.3/train-textcnn.ipynb
# pipeline_v1.3/train-siamese-bilstm-1L.ipynb

python pipeline_v1.3/demo_siamese_bilstm1l_ui.py
# Mở http://127.0.0.1:8000

Chạy lại benchmark v1 / v2

# Sau khi có data/data_ready và artifact trong model/
jupyter notebook model/test-pipeline.ipynb      # v1
jupyter notebook model/test-pipeline-v2.ipynb   # v2 hybrid

Kết luận — Câu chuyện một dự án RAG pháp luật

  1. v1: Bạn xây nền — dữ liệu group-safe, TextCNN định tuyến, benchmark 4 Siamese và chọn LSTM+Cosine làm baseline mạnh nhất trên 8 lớp.
  2. v2: Bạn không dừng ở một pipeline cố định — thử Hybrid và mở rộng topic_topk, chứng minh kết hợp lexical + semantic có thể cạnh tranh TF-IDF full corpus.
  3. v1.3: Bạn chuẩn hoá cho triển khai — gộp lớp hiếm, train lại trên 6 lớp, chọn BiLSTM 1L cho demo, và ship giao diện web + RAG Groq để người dùng thật hỏi đáp.

Lưu ý: metric giữa các phiên bản không so sánh trực tiếp 100% vì khác tập nhãn (8 vs 6 lớp) và khác protocol (model-only vs pipeline end-to-end). Khi báo cáo, nên ghi rõ phiên bản dữ liệu và cấu hình topic_topk.
